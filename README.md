# llama-cpp-cpu-infer
基于 llama.cpp 实现的 C++ 本地大模型 CPU 推理工程

## 项目简介
本项目依托 llama.cpp 最新版 C API，采用 C++17 面向对象方式封装可复用大模型推理类，在 Ubuntu Linux CPU 环境实现 GGUF 量化模型本地离线部署，支持中文标点缓冲流式输出、多轮对话上下文记忆、推理耗时与 Token 吞吐性能指标统计。项目完整记录新版 API 适配、KV 缓存位置冲突、采样器参数迭代等真实踩坑解决方案。
## 技术栈
- 编程语言：C++17
- 构建工具：CMake 4.3.4
- 推理框架：llama.cpp
- 运行系统：Ubuntu Linux
- 编译器：GCC 9.4

## 前置环境准备
1. 拉取 llama.cpp 源码并编译，生成 `.so` 动态链接库；
2. 将 `llama.cpp/build/bin` 目录配置到系统动态链接器，执行 `sudo ldconfig` 更新动态库缓存；
3. 准备 GGUF 量化大模型，本项目测试使用 `qwen2.5-0.5b-instruct-q4_k_m.gguf`。

## 编译 & 运行指令
```bash
# 创建构建目录
mkdir build && cd build
# CMake 配置项目
cmake ..
# 多线程编译
make -j4
# 运行推理程序
./main

## 已实现功能
1.封装LlmInfer推理管理类，统一管控模型、上下文、采样器生命周期，做资源重复释放防护，避免内存异常崩溃；
2.自定义三大结构体：InferParams统一管理所有推理超参数、ChatMsg存储多轮对话消息并自动拼接 Qwen 对话模板、InferResult记录推理耗时、生成 Token 数量、每秒生成 Token 性能数据；
3.适配新版 llama.cpp 四参数采样惩罚接口，有效抑制模型重复生成文本；
4.支持多轮对话上下文缓存，上下文长度溢出自动清空历史会话，保证程序稳定运行；
5.提供单轮对话兼容接口chatOnce，可灵活关闭上下文记忆；
6.实现中文标点缓冲流式输出，优化逐字符打印的卡顿观感；
7.全链路参数合法性校验、空指针判断，异常信息日志打印，提升程序健壮性。

## 项目踩坑记录
1.llama.cpp 持续迭代 C API，旧版单参数重复惩罚接口被废弃，适配新版四参数llama_sampler_init_penalties，解决参数数量不匹配编译报错；
2.新版框架对 KV 缓存序列位置做强连续性校验，多轮对话复用旧上下文会触发解码失败，采用每次推理销毁重建上下文的方案解决该崩溃问题；
3.llama.cpp 编译生成的动态库默认在 build/bin 目录，未配置系统动态链接路径会出现运行时找不到 so 库的问题，通过配置动态链接器 + 刷新缓存解决；
4.长文本多轮对话容易超出上下文窗口上限，设计自动清空会话策略，兼顾对话可用性与程序稳定性；
5.原生逐字符流式输出交互体验差，基于中文标点做缓冲区聚合输出，优化展示效果。
